# Realism Rules

Key Realism Principles:
- Not all tasks are completed
- Some tasks are overdue
- Not every task has an assignee
- Subtasks correlate with parent task state
- Comments are sparse and conversational

Author Decisions:
These rules were derived from common usage patterns in enterprise task management
tools and deliberately enforced in code.

AI Role:
AI was used to review realism assumptions and suggest inconsistencies to avoid.
